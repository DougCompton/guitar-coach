# Repertoire File Format

Store song work in `repertoire.md` beside the roadmaps and daily logs.

Use three lanes:

- `## Learning`
- `## Polishing`
- `## Maintenance`

Each song entry should use this structure:

```markdown
- Song: Title here
  - Status: learning
  - Focus: what must improve right now
  - Current excerpt: verse, chorus, bar 1-4, intro riff, full loop, etc.
  - Notes:
```

## Usage rules

- Prefer one song in `Learning` as the main application target.
- Use `Polishing` for songs that are playable but not stable.
- Use `Maintenance` for older material that should stay alive with occasional review.
- When `Learning` is overloaded, move older unfinished material to `Polishing` or archive it manually.
- During short sessions, pull from `Learning` first.
- During low-energy sessions, pull from `Maintenance` first.
